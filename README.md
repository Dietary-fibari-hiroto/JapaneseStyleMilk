## ブランチ構成図

ブランチは作業スペースのことです！

作業する時は自分のブランチで行ってください！

```bash
-main#本番リリースブランチです。絶対安定してるコードだけ置くようにしてください。CI/CD設定するのでここにpushすると自動的にデプロイ(サーバーに上がる)されます。
  └─ develop#開発途中のコードをまとめておくブランチです。みんなは自分のリポジトリからここにプッシュしていきます。
       ├─ feature/hiroto #ひろとの作業場
       ├─ feature/miki #みきの作業場
       ├─ feature/sena #せなの作業場
       ├─ feature/shuyu #しゅうゆうの作業場
       ├─ feature/rey #レイの作業場
       └─ feature/haruki #はるきの作業場

-feature/setup#ここは以降使いません
```

### gitを使用する際のルール

| **ルール** | 理由 |
| --- | --- |
| 作業は必ず自分の個人ブランチで行ってください。 | mainやdevelopを直接いじると事故る。 |
| タスクに入る前に必ず自分のリポジトリに最新のdevelopをマージしてください。 | コンフリクト(衝突)を避けるため |
| キーやパスワードなどの機密性の高いファイルはgitに上げないでください。 | 流出&悪用の可能性が高いため |
| 一個のプルリクで、提出するタスクは一個だけ | 多いとレビューしづらいし、問題が起きたとき原因追えなくなるから。 |
| タイトルと説明はちゃんと書く | レビューする人が一発で理解できるから。 |

今回個人リポジトリ(feature)からdevelopへのプルリクエストは**タスク単位**で実施する事とします。タスク終了後にdevelopへのプルリクエストを作成するようにしてください。

### コミットメッセージ例:

これはしてくれたら分かりやすくて嬉しいよくらいなので忘れても大丈夫です。

```bash
git commit -m "種類:ざっくり内容"

task: 機能追加、UI構築、ロジック成形
refactor: コード整理（ロジックや動きは変わらない）
fix: バグ修正
test: テストコード追加
chore: 設定ファイルとか環境構築
save: 作業一時中断
```

今回は基本的にgitはターミナル上でコマンドベースで動かします。

- SourceTree（ソースツリー）を導入することで、リポジトリの操作を視覚的に行うことができます。また気になったら聞いてください！

    ## gitをcloneする
    
    クローンは**リモートリポジトリ（ネット上のプロジェクト）を、自分のパソコンにまるごとコピーしてくる作業**のこと！
    
    プロジェクトに参加するためには必須の作業だよ！
    
    ```bash
    #保存したい場所に移動する
    cd ~/~
    #gitをクローンする
    git clone https://github.com/Dietary-fibari-hiroto/JapaneseStyleMilk
    #プロジェクトディレクトリに移動
    cd JapaneseStyleMilk
    ```
    
- ブランチの移動とコミット（タスク開始前と終了後に実施してほしい！）
    
    ## ①`feature/〇〇〇`ブランチに切り替え
    
    ```bash
    #現在滞在しているブランチの確認
    #developで作業してしまっていた...ってことがよくあるので作業前に必ず確認してください。
    git branch
    #末端に「:」が表示されて抜け出せなくなった場合は「q」もしくは[q! + Enter]で抜けましょう。
    
    #ブランチを切り替える
    git checkout feature/〇〇〇
    ```
    
    ## ②最新のdevelopを取り込む(タスク開始前)
    
    タスク開始前に最新の状態を自分の作業ブランチに反映させると、後でマージする時にコンフリクト(衝突)を避けやすくなる！
    
    ```bash
    #ブランチを切り替える
    git checkout feature/〇〇〇
    
    git fetch origin #リモートリポジトリから最新情報を取得
    #ローカルリポジトリ = 自分のパソコンの中
    #リモートリポジトリ = ネット上（GitHub）
    
    git merge origin/develop #developの最新を現在滞在しているブランチに取り込む(マージする)
    
    ```
    
    ## ③feature/〇〇〇の変更後、プルリクエストを送信する方法
    
    1.自分のブランチで作業が終わったら変更を確定させ、その変更をdevelopブランチに反映させるためにマージします！
    
    ```bash
    git add . #コミットする範囲を予約(補足:文末の「.」はすべてを意味します。特定のファイルだけコミットしたいときはそのファイル名が入るんだけど、基本的にすべてで大丈夫だお)
    
    git commit -m "やった作業を簡単にメモ" #予約した変更をまとめて保存
    #ここで自分のローカルリポジトリからリモートリポジトリへアップできた状態
    #※ここでコミットしてないと、ブランチを移動する時に「作業中の変更があります！」って怒られる。
    
    git push origin feature/〇〇〇 #リモートサーバー（GitHub）へ変更を送信する
    
    ```
    
    2.次にGitHub上でプルリクエストを作ります
    
    GitHubを開くと、「Compare & pull request」って緑色のボタンが出るはず！ 押すとプルリクエスト作成画面になる。
    
    もし出ないときは：
    
    - 対象リポジトリに行って
    - 「Pull requests」タブをクリック
    - 「New pull request」を押す
    
    そこで画面上部の
    
    - **ベース（base）** → マージ先（develop ）
    - **比較（compare）** → 自分のブランチ（例: `feature/〇〇〇`）
    を選ぶ！
    
    3.プルリクエストのタイトル(タスク名or番号)と説明を書く
    
    - **タイトル-タスク名**：何をしたか一言で
    - **説明**：どう動くか、を簡単に書く
    → レビューしてくれる人が分かりやすいように！
    
    4. 「Create pull request」ボタンを押す
    
    これでプルリクエストが正式に作成される！
    
    ## ④もしタスクが終わりきらずに作業を終える場合
    
    この場合は自分のブランチにだけコミット,プッシュします。
    
    パソコンが壊れて使えなくなったとしてもsaveしておけば別のパソコンでも続きから作業ができるので一応保存しておくっていうイメージ
    
    ```bash
    git add . 
    
    git commit -m "save:やった作業を簡単にメモ" 
    
    git push origin feature/〇〇〇 
    ```
    
    ### ***この状態で作業を再開する時は②の作業は不要です!***
    
    何もせずにパソコン立ち上げてそのまま再開してもろて
    
- 機密性の高いファイルの扱い方と.gitignore
    
    外に漏れてはいけない情報を安全に管理するために、機密性の高い情報はgit外で共有します。(共有できるならdiscord,notion等でもOK!)
    
    今回gitに上げないファイルの一例:
    
    - **パスワードとかAPIキー**が入ってるファイル
    - **ビルドしてできるゴミファイル**（例：`node_modules/`とか）
    - **自分だけの設定ファイル**（例：VSCodeの`.vscode/settings.json`
    
    下二つは上げてしまうとgitのデータが大きくなってしまうので各自のローカルリポジトリで管理します。
    
    ## .gitignore
    
    .gitignoreは**Gitに「このファイルはアップしないで！」って教えるための設定ファイル**のこと。ここ記述しておくと特定のファイルのコミットを無視することができます。
    
    アップしたくないファイルがある場合はここに記述しましょう。
    
    場所はプロジェクトの一番上の階層にいます
